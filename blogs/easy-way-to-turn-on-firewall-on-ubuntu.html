An easy firewall configuration for Ubuntu

To configure a firewall on an Ubuntu server so that only HTTPS (port 443) and SSH (port 22) ports are open, and all other ports are blocked, you can use `ufw` (Uncomplicated Firewall), which is a user-friendly front-end for managing iptables firewall rules. Here's how you can do it:

  • Install UFW: If `ufw` is not already installed on your Ubuntu server, you can install it by running:
    sudo apt update sudo apt install ufw
  • Enable UFW: Before enabling `ufw`, ensure that the SSH port is allowed; otherwise, you might be locked out of your server. To allow SSH (port 22) connections, run:
    sudo ufw allow ssh
    Or you can explicitly specify the port number:
    sudo ufw allow 22/tcp
  • Allow HTTPS: To allow HTTPS (port 443) connections, run:
    sudo ufw allow https
    Or you can explicitly specify the port number: sudo ufw allow 443/tcp
  • Enable UFW: After configuring the rules, enable `ufw` by running:
    sudo ufw enable
    This command will apply the rules and start the firewall. You'll be asked to confirm the operation; answer `y` to proceed.
  • Check UFW Status: You can check the status of `ufw` and see the rules that are currently applied by running:
    sudo ufw status
    This command shows a list of allowed and denied connections.
  • Denying All Other Traffic (Optional): By default, UFW is set to deny all incoming connections and allow all outgoing connections. If you want to explicitly deny all other incoming traffic not specified in your rules, you don't need to do anything additional since this is the default behavior. However, for outgoing connections, if you wish to change the default policy, you can do so by running: sudo ufw default deny incoming sudo ufw default allow outgoing Note that these commands set the default policies and do not need to be executed if you're okay with the default settings of denying all incoming (not explicitly allowed) and allowing all outgoing connections.
  • Additional Rules: If you need to open more ports in the future, you can add new rules using the sudo ufw allow> command followed by the service name or port number. Make sure you replace `ssh` and `https` with specific port numbers if your services run on non-standard ports. Always be cautious when modifying firewall settings to avoid locking yourself out of the server.
