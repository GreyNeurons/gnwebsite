Windows Subsystem for Linux (WSL) – Detailed Summary

✅ What is WSL?

WSL (Windows Subsystem for Linux) is a compatibility layer for running Linux directly on Windows without using a virtual machine (VM) or dual booting. It provides a native Linux experience within the Windows environment, enabling users to run Linux distributions, execute shell commands, develop software, and access Linux tools seamlessly.

  • WSL1: A lightweight translation layer that converts Linux system calls to Windows NT kernel calls.
  • WSL2: Uses a real Linux kernel inside a virtualized lightweight VM, offering better performance and full system call compatibility.

✅ Why Use WSL?

WSL is useful for developers, sysadmins, and power users who need Linux tools but prefer to work within Windows. Benefits include:

  • Seamless Linux Integration – Run Linux commands alongside Windows applications.
  • Developer Productivity – Use Linux-exclusive development tools like GCC, Python, Node.js.
  • Performance – WSL2 provides near-native Linux performance.
  • File System Interoperability – Access Windows files from Linux and vice versa.
  • Security & Networking – Run security tools, test network configurations, and use SSH.
  • Low Overhead – Unlike full VMs, WSL is lightweight and efficient.

✅ How Does WSL Work?

  1. Install WSL: Run wsl --install in PowerShell and install a Linux distro from the Microsoft Store.
  2. Choose Between WSL1 and WSL2: WSL1 is lightweight, while WSL2 supports Docker.
  3. Run Linux Commands in Windows: Use Windows Terminal, PowerShell, or CMD.
  4. Interoperability with Windows: Access files between Linux and Windows seamlessly.
  5. Networking & Development: Supports Docker, Kubernetes, SSH, and cloud-native tools.

✅ When Was WSL Introduced?

  • 2016 – WSL1 introduced as a translation layer for Linux system calls.
  • 2019 – WSL2 launched with a real Linux kernel.
  • 2021 – Windows 11 introduced simplified installation and GUI support.

✅ Where Can You Use WSL?

WSL is available on:

  • Windows 10 (1903+) – Limited to WSL1 in older versions, WSL2 requires 2004+.
  • Windows 11 – Best WSL experience with built-in support.
  • Windows Server 2022 – Enterprise use cases.

WSL Configuration & Optimization Summary

Comprehensive Guide for Performance Optimization

1️⃣ WSL System Information

wsl --version
WSL version: 2.4.10.0
Kernel version: 5.15.167.4-1
Windows version: 10.0.19045.5371

Installed WSL Distros:

wsl --list --verbose
NAME            STATE           VERSION
* Ubuntu-22.04  Stopped         2
  Ubuntu        Stopped         2

2️⃣ WSL Resource Configuration

WSL Configuration (.wslconfig)

[wsl2]
memory=8GB
processors=4
swap=45GB
swapFile=C:\\wsl-swap.vhdx

To apply changes:

wsl --shutdown
wsl

3️⃣ Swap Configuration & Fixing Issues

If WSL swap creation fails with "Access Denied" error, run:

fsutil file createnew C:\wsl-swap.vhdx 48318382080
icacls C:\wsl-swap.vhdx /grant "NT AUTHORITY\\SYSTEM:(F)"

4️⃣ Benchmarking WSL Swap Performance

Run the following command inside WSL:

dd if=/dev/zero of=/mnt/c/wsl-swap-benchmark bs=1G count=1 oflag=direct
dd if=/mnt/c/wsl-swap-benchmark of=/dev/null bs=1G count=1 iflag=direct

5️⃣ Stress Testing CPU & RAM

sudo apt update && sudo apt install stress -y
stress --cpu 4 --vm 2 --vm-bytes 12G --timeout 60s

6️⃣ Performance Observations

  • More Available RAM: 3.7GB (previously 1.2GB)
  • Better Swap Performance: ~1000-1500 MB/s
  • Improved System Responsiveness: Less sluggish under load
