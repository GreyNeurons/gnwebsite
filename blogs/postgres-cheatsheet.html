# **PostgreSQL Cheatsheet for Ubuntu WSL** ## **1ï¸âƒ£ Check PostgreSQL Status** ### **Check if PostgreSQL is Running** ```bash sudo systemctl status postgresql ``` - `active (running)`: PostgreSQL is working fine âœ… - `active (exited)`: PostgreSQL is running but systemd exited (normal in WSL) âœ… - `failed`: Check logs to debug âŒ ### **Check Running PostgreSQL Processes** ```bash pgrep -fa postgres ``` - Shows active PostgreSQL processes. ### **Check PostgreSQL Port Listening** ```bash ss -tulnp | grep postgres ``` - Should return `5432` (default PostgreSQL port). ## **2ï¸âƒ£ Start, Stop, and Restart PostgreSQL** ```bash sudo systemctl start postgresql sudo systemctl stop postgresql sudo systemctl restart postgresql ``` ### **Enable Auto-Start on Boot** ```bash sudo systemctl enable postgresql ``` ## **3ï¸âƒ£ PostgreSQL Configuration & Security** ### **Locate Config Files** ```bash sudo find /etc/postgresql/ -name postgresql.conf ``` Default locations: - `postgresql.conf`: `/etc/postgresql/17/main/postgresql.conf` - `pg_hba.conf`: `/etc/postgresql/17/main/pg_hba.conf` ### **Change Memory & Performance Settings (`postgresql.conf`)** ```ini # Prevent PostgreSQL from using too much RAM work_mem = 8MB shared_buffers = 512MB maintenance_work_mem = 64MB temp_buffers = 8MB ``` ### **Apply Configuration Changes** ```bash sudo systemctl restart postgresql ``` Check applied settings inside `psql`: ```sql SHOW work_mem; SHOW shared_buffers; SHOW maintenance_work_mem; ``` ### **Check Logs for Issues** ```bash sudo journalctl -u postgresql --no-pager | tail -20 sudo tail -f /var/log/postgresql/postgresql-17-main.log ``` ## **4ï¸âƒ£ Secure PostgreSQL Access (`pg_hba.conf`)** ### **Check Authentication Rules** ```bash sudo nano /etc/postgresql/17/main/pg_hba.conf ``` Default security settings (safe for local use): ```ini local all all peer host all all 127.0.0.1/32 scram-sha-256 host all all ::1/128 scram-sha-256 ``` ðŸš¨ **To allow remote access:** 1. Edit `postgresql.conf`: ```ini listen_addresses = '*' ``` 2. Edit `pg_hba.conf` (allow specific IPs only!): ```ini host all all 192.168.1.0/24 scram-sha-256 ``` 3. Restart PostgreSQL: ```bash sudo systemctl restart postgresql ``` ## **5ï¸âƒ£ PostgreSQL User & Database Management** ### **Switch to PostgreSQL Shell** ```bash sudo -i -u postgres psql ``` Or directly as `rajesh`: ```bash psql -U rajesh -d postgres ``` ### **List Users** ```sql SELECT usename FROM pg_user; ``` ### **Create a New User** ```sql CREATE USER rajesh WITH SUPERUSER CREATEDB CREATEROLE ENCRYPTED PASSWORD 'your_secure_password'; ``` ### **Change Password for `postgres`** ```sql ALTER USER postgres WITH PASSWORD 'new_password'; ``` ### **Create a Database Owned by `rajesh`** ```sql CREATE DATABASE mydb OWNER rajesh; ``` ### **List Databases** ```sql \l ``` ### **Connect to a Database** ```bash psql -U rajesh -d mydb ``` ### **Verify Current User & Database** ```sql SELECT current_user, current_database(); ``` ### **Exit `psql`** ```sql \q ``` ## **6ï¸âƒ£ PostgreSQL Monitoring & Performance** ### **Check Active Connections** ```sql SELECT * FROM pg_stat_activity; ``` ### **Show PostgreSQL Version** ```sql SELECT version(); ``` ### **Enable Query Logging (Optional, in `postgresql.conf`)** ```ini log_statement = 'all' ``` Restart PostgreSQL to apply changes: ```bash sudo systemctl restart postgresql ``` ## **7ï¸âƒ£ Backup & Restore PostgreSQL** ### **Backup a Database** ```bash pg_dump -U rajesh mydb > mydb_backup.sql ``` ### **Restore a Database** ```bash psql -U rajesh -d mydb -f mydb_backup.sql ``` ### **Backup All Databases** ```bash pg_dumpall -U rajesh | gzip > /var/backups/postgresql_$(date +%F).sql.gz ``` --- ### **ðŸš€ Best Practices** âœ… **Use `rajesh` instead of `postgres` for daily work** âœ… **Keep `listen_addresses` restrictive (`localhost` or specific IPs)** âœ… **Use `scram-sha-256` authentication** for security âœ… **Optimize memory settings** to balance PostgreSQL & local AI models âœ… **Monitor with `pg_stat_activity` to find slow queries** âœ… **Automate backups using `pg_dump` or `pg_dumpall`** --- ðŸ’¡ **Now PostgreSQL is secure, optimized, and ready for your Java Q&A app! ðŸš€** Let me know if you need further tweaks! ðŸŽ¯
